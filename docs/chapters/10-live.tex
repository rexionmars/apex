\chapter{Live monitor}

\eyebrow{real-time fat overlay for framing}

\vspace{4pt}
The \textbf{Live monitor} shows a fat overlay in real time, from a camera or a
video file, as an aid to framing the carcass during capture. It is a
\emph{framing guide}, not the official measurement.

\section{Operation}

\shot{live-monitor.jpeg}{The Live monitor, stopped. A source switch (Live camera /
Video file) chooses the frame source; \tele{Start} begins the overlay and
\tele{Capture background} records the empty scene for subtraction. The engine
state and device are shown at the top.}{live-monitor}

\begin{enumerate}[leftmargin=1.4em,itemsep=3pt]
  \item The source is selected: \textbf{Live camera} or \textbf{Video file}.
  \item The \tele{Start} control begins playback. The video plays and the fat
        overlay (cyan) tracks it, with a live FPS and fat-percentage readout in
        the corner.
  \item For a fixed camera, \tele{Capture background} records the empty scene; the
        application subtracts it from each frame so the overlay is restricted to
        the carcass. The carcass is then positioned.
\end{enumerate}

\section{How it works, and its limits}

The monitor uses classical background subtraction ($\sim$10\,ms) plus the fat
model at reduced resolution, reaching roughly real-time rates on the tested
hardware. It does \emph{not} use the more accurate but slower background removal
of the Analysis pipeline.

\begin{caution}[title={Framing guide, not the validated number}]
The live overlay is approximate, a visual aid for framing. The validated fat
number comes from the Analysis tab, run on a captured image with the more accurate
background removal.
\end{caution}
